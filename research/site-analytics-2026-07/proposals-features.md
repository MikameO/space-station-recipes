# Каталог продуктовых идей — 13 предложений с оценками

**Дата:** 2026-07-12. Источник: брейншторм-сессия по вопросу «как сделать сайт полезнее/интереснее/проще/ценнее для игроков SS14», скорректированный анализом Метрики ([insight-metrika-audience.md](insight-metrika-audience.md)).
**Оценки:** HAE = часы фокусного внимания человека (методика проекта). Маппинг Oxx → возможности из [../ss14-community-demand/SUMMARY.md](../ss14-community-demand/SUMMARY.md).

**Ключевая рамка:** сайт организован «от реагента» (нужно знать, что ищешь), а игрок думает «от задачи» («пациент горит — что колоть?», «что будет, если смешать?», «что наварить к смене?»). Самые ценные идеи — развороты уже существующих данных под игровую ситуацию: `data.json` уже содержит `effectTags`, `metabolismPaths`, `accessibility`, `verifiedMechanics`, `plantEffects` — многим фичам ниже экстрактор не нужен вообще, только фронтенд.

---

## Ось «Полезнее» — закрыть задачные сценарии

### 1. «Чем лечить?» — медицинский режим ⭐ фаворит по value/effort
Выбор типа урона (Brute/Burn/Toxin/Airloss/генетика/радиация) + форк → ранжированный список препаратов: лечение за юнит, порог OD, доступность (диспенсер → варка → ботаника). Расширяет аудиторию с химиков на **всех медиков** (их на станции в разы больше). Данные есть. **2–4 HAE, чисто фронтенд.** → O2+O6.

### 2. Симулятор стакана («что будет, если смешать»)
Ввод фактического содержимого стакана с объёмами → прогон реакций с каскадами промежуточных продуктов → результат с предупреждениями («на шаге 8 будет дым»). Отличие от «What Can I Make?»: тот про доступные рецепты, этот про физику смешения. Боль каждого новичка. Дифференциатор, который вики/таблицы не повторят. **4–8 HAE.** → O1 (Tier 1 спроса).

### 3. Пресеты «начало смены»
Кнопки «набор медхимика / ботаника / бар-стартер» по форку → one-click в существующий Batch Planner (механика `loadStrategyIntoBatch` уже есть для антаг-стратегий — добавить мирные пресеты). **1–2 HAE** (в основном кураторство). → O7.

### 4. Fork diff («переехал на Goob — что поменялось?»)
Таблица added/removed/modified между двумя форками. Вся инфраструктура (first-wins merge, `forkStatus`, auto-diff) уже вычисляет это — нужен UI-разворот. Вики форков хронически отстают. **~2 HAE.** → O8, O3.

### 5. Режим «во время игры» (second-screen)
Cook-mode: пошаговый чеклист рецепта с чекбоксами, крупный шрифт, прогресс в localStorage — под телефон/второй монитор. Плюс дожать PWA: manifest.json есть, добавить service worker + офлайн-кеш data.json. ⚠ vis-network грузится с unpkg — для офлайна захостить локально. Метрика подтверждает: ядро УЖЕ использует сайт так. **2–4 HAE.** → O1/O7.

## Ось «Проще» — снизить порог входа

### 6. Ролевые «линзы» на старте
Первый визит: «Я химик / медик / ботаник / бармен / антагонист» → преднастроенные фильтры + нужная вкладка. 8 табов пугают новичка; туториал уже есть, линзы — следующий шаг. **~2 HAE.**

### 7. Русский поиск (без полной локализации)
Двуязычный поисковый индекс: FTL-локали RU-форков уже содержат переводы имён реагентов, экстрактор умеет `load_all_localization`. Игрок ищет «трикордразин» → находит Tricordrazine. **~2 HAE** (полная локализация UI — 8+). → O11. Приоритет понижен данными Метрики (RU ~9%), но остаётся ставкой роста — см. RU-парадокс в insight-файле.

## Ось «Интереснее» — удержание и виральность

### 8. Тренажёр-квиз
«Угадай рецепт», «что получится при смешении», таймер, серия из 10, шаринг результата. Контент генерится из данных бесконечно. Хорошо расходится по Discord. **~4 HAE.**

### 9. «Химический changelog для игроков»
Автодифф data.json между регенерациями: «в vanilla изменили X, на Corvax добавили Y». Git-история репозитория уже хранит все прошлые data.json — источник бесплатный. Никто в экосистеме этого не делает, вики отстают. Готовый контент для Discord-анонсов. **2–4 HAE.** → O3+O8 (свежесть данных = конкурентное преимущество №1 по demand-study).

### 10. Песочница-стакан (drag & drop)
Визуальный beaker с анимацией результата (цвет/дым/взрыв) — обучающая игрушка поверх симулятора (№2). Самая залипательная и самая дорогая. **8+ HAE. Делать только после №2 как его обёртку.**

## Ось «Ценнее» — каналы дистрибуции

### 11. Discord-бот
`!chem tricordrazine` → карточка + ссылка. Discord — главный хаб серверов; каждый вопрос новичка в канале → визит. data.json версионирован (schemaVersion) — по сути готовый API. **~4 HAE + хостинг** (⚠ выводит проект за рамки статики: поддержка). → O4.

### 12. Steam-гайды ×N
Канал уже работает (25% визитов, 3.7% отказов — см. insight). Отдельные гайды по ролям (медхимия/ботаника/антаг) с deep-links (share-URL уже кодируют состояние). **1–2 HAE/гайд, почти без кода.** → O4/O7.

### 13. Партнёрства с RU-форками
Ссылка в вики/MOTD/пине Discord у ADT/Sunrise/Corvax — дешевле любого SEO. Аргумент: единственный справочник, учитывающий именно их изменения (lineage, blocked reactions, per-fork difficulty). **~1 HAE переговоров.** → O4+O11.

## Инфраструктурные (не фичи, но всплыли)

- **SEO pre-render** — генерить `/reagent/<id>.html` (953 стр.) тем же Python-пайплайном: title, рецепт, эффекты, canonical, «открыть в ChemDB», + sitemap. Чинит мёртвый канал (6 визитов из поиска за квартал). **2–4 HAE.** → O4. По синтезу двух исследований — приоритет №1.
- **data.json 2.4MB и растёт** с каждым форком — ленивая загрузка секций (sources, antag) для мобильного сценария.
- **Событийная аналитика** — ✅ сделано 2026-07-12, см. [handoff-analytics-goals.md](handoff-analytics-goals.md).

## Отвергнуто / отложено в сессии

- Полная RU-локализация UI сразу — данные Метрики не подтверждают приоритет; сначала RU-поиск + RU-каналы (№7, №13).
- Ачивки/геймификация в лоб — не подтверждено болью; квиз (№8) покрывает «интереснее» дешевле.
- User-submitted стратегии с голосованием — модерационная нагрузка; текущий GitHub-flow (issue-шаблоны с обязательным пруфом) достаточен.

## Рекомендованная последовательность (синтез, 2026-07-12)

1. SEO pre-render → 2. Steam-гайды → 3. дождаться данных целей Метрики (2–3 нед.) → 4. «Чем лечить?» → 5. пресеты/cook-mode → 6. симулятор стакана. RU-трек (№7+№13) — параллельно, малыми силами. Полная таблица с обоснованием — в [SUMMARY.md](SUMMARY.md) §2.
